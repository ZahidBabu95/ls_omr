# 🚀 AmarSchool OMR: Project Roadmap & Next Steps

ল্যান্ডিং পেজের ডাইনামিক ডেটা, অ্যাডমিন কন্ট্রোল প্যানেল (Pricing, Features, OMR Templates, Downloads, Settings), এবং cPanel লাইভ ডেপ্লয়মেন্ট সফলভাবে সম্পন্ন হয়েছে। প্রজেক্টের "মার্কেটিং ও প্রেজেন্টেশন (Phase 1)" ধাপ শেষ। 

এখন প্রজেক্টকে একটি সম্পূর্ণ SaaS (Software as a Service) মডেলে রূপান্তর করতে নিচের পদক্ষেপগুলো (Phase 2 & 3) নেওয়া উচিত:

## ১. Authentication ও Role Management (অত্যন্ত জরুরি)
বর্তমানে ডিফল্ট লারাভেল ব্রীজ (Breeze) এর রেজিস্ট্রেশন পেজ (`/register`) সকলের জন্য ওপেন আছে, যার মাধ্যমে যেকেও অ্যাডমিন ড্যাশবোর্ডে প্রবেশ করতে পারে। 
- **করণীয়:** `Spatie/Laravel-Permission` প্যাকেজ ইন্সটল করে Super Admin এবং School Admin এর জন্য আলাদা Role তৈরি করা। সুপার অ্যাডমিনের ড্যাশবোর্ড এবং ক্লায়েন্ট স্কুলের ড্যাশবোর্ড সম্পূর্ণ আলাদা করা। 

## ২. Subscription ও Payment Gateway Integration
যেহেতু ল্যান্ডিং পেজে এখন ডাইনামিক প্রাইসিং প্ল্যান (Pricing Plans) শো করানো হয়েছে, পরবর্তী ধাপে এটিকে পেমেন্ট গেটওয়ের সাথে যুক্ত করতে হবে।
- **করণীয়:** bKash, SSLCommerz অথবা aamarpay এর মত অটোমেটেড পেমেন্ট গেটওয়ে যুক্ত করা। কোনো স্কুল যখন "Growth" বা "Pro" প্যাকেজ কিনবে, তখন পেমেন্ট কনফার্ম হওয়ার সাথে সাথে তাদের অ্যাকাউন্ট অটোমেটিকভাবে আপগ্রেড হয়ে যাবে।

## ৩. License Management System (API)
আপনার প্রজেক্টে একটি ডেস্কটপ সফটওয়্যার (LSOMR_Setup.exe) আছে। সফটওয়্যারটি ডাউনলোড করার পর যেন যেকেউ ফ্রি বা লাইফটাইম অ্যাক্সেস না পায়, সেজন্য লাইসেন্সিং সিস্টেম করতে হবে।
- **করণীয়:** অ্যাডমিন প্যানেল থেকে প্রতিটি স্কুলের জন্য License Key বা API Token জেনারেট করে দেওয়া। ডেস্কটপ সফটওয়্যারটি রান করার সময় প্রথমে সেই API Token ভেরিফাই করবে। 

## ৪. Core OMR Evaluation Engine (স্কুল ড্যাশবোর্ড)
সবচেয়ে গুরুত্বপূর্ণ লজিক্যাল কাজ। স্কুলগুলোর লগইন করার পর নিজস্ব ডেডিকেটেড ড্যাশবোর্ড (Multi-tenant architecture) তৈরি করা।
- **করণীয়:** স্টুডেন্ট ডাটাবেস ম্যানেজমেন্ট, এক্সাম বা পরীক্ষা তৈরি করা, এবং ডেস্কটপ/মোবাইল স্ক্যানার থেকে সরাসরি সার্ভারে API এর মাধ্যমে স্ক্যান করা OMR শিটের ডাটা ও রেজাল্ট প্রসেস করে পুশ করা। 

## ৫. Cloud Storage Integration (AWS S3 / Cloudflare R2)
ভবিষ্যতে যখন হাজার হাজার OMR শিটের ছবি সার্ভারে আপলোড হতে শুরু করবে, তখন cPanel এর লোকাল স্টোরেজ দ্রুত ফুল হয়ে যাবে বা স্লো হয়ে পড়বে।
- **করণীয়:** `.env` এ AWS / R2 এর API Key কনফিগার করে স্টোরেজ ড্রাইভ হিসেবে `s3` ব্যবহার করা, যেন সমস্ত ভারী ফাইল ক্লাউডে সেভ হয়।

## ৬. Analytics / Reports Dashboard
সুপার অ্যাডমিন ড্যাশবোর্ডে ওয়েবসাইটের ট্রাফিক, কতগুলো ডেমো রিকোয়েস্ট আসলো, কতগুলো স্কুল সাইনআপ করলো এবং মোট আয় কত হলো, তা সুন্দর চার্ট বা গ্রাফ (Chart.js বা ApexCharts) এর মাধ্যমে শো করানো।

---
**পরবর্তী কাজের শুরু:** আপনি চাইলে সিকিউরিটির জন্য প্রথমে *Authentication এবং Role Management* এর কাজ শুরু করতে বলতে পারেন, অথবা সরাসরি *School Dashboard* লজিকের কাজ শুরু করতে পারেন।
